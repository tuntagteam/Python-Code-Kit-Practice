## 1. ระบบดุลสมการพหุนามระดับสูง (Polynomial Fast Evaluation)

### รายละเอียดโจทย์

ในการคำนวณกราฟิกสามมิติระดับลึก เราต้องประเมินค่าของสมการพหุนาม $P(x) = a_n x^n + a_{n-1} x^{n-1} + \dots + a_1 x + a_0$ ณ จุด $x$ ที่กำหนดให้จำนวนหลายล้านครั้ง

หากใช้การคำนวณยกกำลังแบบตรงๆ ($x^i$) จะทำงานช้าเกินไปและเสี่ยงต่อปัญหา Overflow จงเขียนฟังก์ชันประมวลผลที่มีประสิทธิภาพสูงโดยใช้อัลกอริทึม **Horner's Method** เพื่อหาผลลัพธ์ของสมการพหุนามภายใต้การดำเนินการมอดุโล ($10^9 + 7$) เพื่อควบคุมขนาดข้อมูล โดยห้ามใช้โอเปอเรเตอร์ยกกำลัง  และห้ามใช้ฟังก์ชันสำเร็จรูปใดๆ

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็ม $n$ ($1 \le n \le 10^5$) แทนดีกรีสูงสุดของพหุนาม และจำนวนเต็ม $x$ ($0 \le x \le 10^9$) แทนค่าวิกฤตที่จะนำไปแทนในสมการ
* **บรรทัดที่สอง:** จำนวนเต็ม $n+1$ ตัวแยกด้วยช่องว่าง แทนสัมประสิทธิ์เรียงลำดับจาก $a_n, a_{n-1}, \dots, a_1, a_0$ ($0 \le a_i \le 10^9$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงค่า $P(x) \pmod{10^9 + 7}$

### ตัวอย่าง

**Input**

```
3 2
2 0 -5 3

```

**Output**

```
9

```

**คำอธิบาย:** สมการคือ $P(x) = 2x^3 + 0x^2 - 5x + 3$ แทนค่า $x = 2$ จะได้ $2(8) + 0(4) - 5(2) + 3 = 16 + 0 - 10 + 3 = 9$

## 2. ระบบค้นหาจุดตัดยอดเขา (Longest Mountain Subarray)

### รายละเอียดโจทย์

นักภูมิศาสตร์ดิจิทัลต้องการประมวลผลข้อมูลความสูงของภูเขาที่เรียงตัวกันเป็นอาร์เรย์ยาว $n$ ช่อง พวกเขาต้องการค้นหาช่วงของข้อมูลที่ยาวที่สุดที่มีลักษณะเป็น **"รูปทรงภูเขา (Mountain Shape)"**

อาร์เรย์ย่อย $A[i \dots j]$ จะถือว่าเป็นทรงภูเขาก็ต่อเมื่อ:

1. มีความยาวอย่างน้อย 3 ช่อง ($j - i + 1 \ge 3$)
2. มีดัชนีจุดยอด $k$ ($i < k < j$) ที่ทำให้ข้อมูลฝั่งซ้ายขยับขึ้นอย่างเข้มงวด: $A[i] < A[i+1] < \dots < A[k]$ และข้อมูลฝั่งขวาลดลงอย่างเข้มงวด: $A[k] > A[k+1] > \dots > A[j]$

จงเขียนโปรแกรมที่มีความเร็วระดับ $O(n)$ และใช้หน่วยความจำเพิ่ม $O(1)$ เพื่อหาความยาวของภูเขาที่ยาวที่สุดนี้

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็ม $n$ ($3 \le n \le 10^5$) แทนจำนวนจุดวัดความสูง
* **บรรทัดที่สอง:** จำนวนเต็ม $n$ ตัวแยกกันด้วยช่องว่าง ($0 \le \text{ความสูง} \le 10^6$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงความยาวที่มากที่สุดของกลุ่มข้อมูลที่เป็นรูปทรงภูเขา (หากไม่พบเลยให้ตอบ 0)

### ตัวอย่าง

**Input**

```
8
2 1 4 7 3 2 5 1

```

**Output**

```
5

```

**คำอธิบาย:** ช่วงภูเขาที่ยาวที่สุดคือ [1, 4, 7, 3, 2] มีดัชนีจุดยอดคือ 7 ความยาวรวมเท่ากับ 5 ช่อง

## 3. ระบบวิเคราะห์ความผันผวนแบบกระจาย (Inversion Count)

### รายละเอียดโจทย์

ในการวัดระดับความไม่เป็นระเบียบของชุดข้อมูลอันดับ (Ranking Quality) เราต้องการหาค่า **Inversion Count** ของอาร์เรย์ตัวเลข $A$ ขนาด $n$ ช่อง

สภาวะ Inversion จะเกิดขึ้นเมื่อมีดัชนีคู่ใดๆ $(i, j)$ ที่ $i < j$ แต่ดันมีค่า $A[i] > A[j]$ (ตัวเลขที่มากอยู่ก่อนหน้าตัวเลขที่น้อยกว่า) จงคำนวณหาจำนวนคู่ Inversion ทั้งหมดในอาร์เรย์ โดยระบบต้องทำงานได้เร็วกว่าการลูปเช็กแบบตรงๆ (ห้ามใช้ $O(n^2)$ เด็ดขาด ให้ใช้แนวคิดดัดแปลงจาก Divide and Conquer ด้วยความเร็ว $O(n \log n)$)

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็ม $n$ ($1 \le n \le 10^5$)
* **บรรทัดที่สอง:** จำนวนเต็ม $n$ ตัวแยกด้วยช่องว่าง ($1 \le A[i] \le 10^6$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงจำนวนคู่ Inversion ทั้งหมด

### ตัวอย่าง

**Input**

```
5
2 4 1 3 5

```

**Output**

```
3

```

**คำอธิบาย:** มีคู่ที่เกิดการสลับลำดับทั้งหมด 3 คู่ ได้แก่ $(2, 1)$, $(4, 1)$, และ $(4, 3)$

## 4. ระบบสถานีสัญญาณเครือข่ายไร้สาย (Sliding Window Maximum)

### รายละเอียดโจทย์

สถานีส่งสัญญาณกำลังเคลื่อนที่ไปตามรางเส้นตรงที่มีเซนเซอร์ตรวจจับติดตั้งอยู่ $n$ จุด ตัวสถานีจะมีเสารับสัญญาณที่ครอบคลุมพื้นที่หน้ากว้างขนาด $k$ ช่อง (Sliding Window of Size $k$)

เมื่อสถานีขยับจากซ้ายไปขวาข้ามเซนเซอร์ทีละช่อง ระบบคอมพิวเตอร์ส่วนกลางต้องการทราบ **ค่าความแรงสัญญาณที่มากที่สุด** ที่อยู่ภายในหน้ากว้าง $k$ ช่อง ณ เวลานั้นเสมอ จงประมวลผลเพื่อหาลิสต์ของค่าสูงสุดในแต่ละหน้ากว้างด้วยความเร็วเฉลี่ย $O(n)$ โดยใช้โครงสร้างข้อมูลประเภท Monotonic Queue ขยับรับส่งข้อมูลเข้าออก

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็มสองตัว $n$ และ $k$ ($1 \le k \le n \le 10^5$) แทนจำนวนเซนเซอร์และหน้ากว้างของเสาสัญญาณ
* **บรรทัดที่สอง:** จำนวนเต็ม $n$ ตัวแยกด้วยช่องว่าง แทนค่าความแรงสัญญาณของเซนเซอร์แต่ละจุด

### ข้อมูลส่งออก (Output)

* ชุดจำนวนเต็มแยกกันด้วยช่องว่างในบรรทัดเดียว แสดงค่าสูงสุดของแต่ละหน้ากว้างตามลำดับจากซ้ายไปขวา

### ตัวอย่าง

**Input**

```
8 3
1 3 -1 -3 5 3 6 7

```

**Output**

```
3 3 5 5 6 7

```

**คำอธิบาย:**

* Window 1: [1, 3, -1] -> max = 3
* Window 2: [3, -1, -3] -> max = 3
* Window 3: [-1, -3, 5] -> max = 5
* Window 4: [-3, 5, 3] -> max = 5
* Window 5: [5, 3, 6] -> max = 6
* Window 6: [3, 6, 7] -> max = 7

## 5. ระบบคำนวณการเติบโตแบบก้าวกระโดด (Matrix Exponentiation)

### รายละเอียดโจทย์

นักคณิตศาสตร์ต้องการคำนวณหาค่าในลำดับฟีโบนัชชี (Fibonacci Number) ที่ตำแหน่งเร้นลับระดับมหาศาล $F_n$ โดยที่ $n$ มีค่าได้สูงสุดถึง $10^{18}$ แน่นอนว่าการใช้ Dynamic Programming ลูปบวกเพิ่มไปทีละตัวจะติดปัญหา Time Limit Exceeded ทันที

จงเขียนโปรแกรมหาค่า $F_n \pmod{10^9 + 7}$ โดยใช้เทคนิค **Matrix Exponentiation** (การยกกำลังเมทริกซ์อย่างรวดเร็วแบบรันไทม์ $O(\log n)$) ร่วมกับความสัมพันธ์เชิงเส้น:

$$\begin{bmatrix} F_{n+1} \\ F_n \end{bmatrix} = \begin{bmatrix} 1 & 1 \\ 1 & 0 \end{bmatrix}^n \begin{bmatrix} F_1 \\ F_0 \end{bmatrix}$$

### ข้อมูลนำเข้า (Input)

* จำนวนเต็ม 1 บรรทัด คือค่า $n$ ($0 \le n \le 10^{18}$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงค่า $F_n \pmod{10^9 + 7}$ (กำหนดให้ $F_0 = 0, F_1 = 1$)

### ตัวอย่าง

**Input**

```
10

```

**Output**

```
55

```

## 6. ระบบตัดสายใยแก้วนำแสง (Segment Tree - Point Update, Range Query)

### รายละเอียดโจทย์

ในโครงข่ายสายใยแก้วนำแสงยาว $n$ กิโลเมตร (จุดวัดพิกัด $1$ ถึง $n$) แต่ละพิกัดมีค่าอัตราการรับส่งข้อมูลเริ่มต้นเป็น 0 ระบบนี้ต้องการโครงสร้างข้อมูลที่สามารถทำงานได้สองประเภทอย่างรวดเร็วในเวลา $O(\log n)$:

1. 1 idx val: อัปเดตข้อมูล (Point Update) เพิ่มค่าสัญญาณ ณ พิกัด idx ด้วยค่า val
2. 2 L R: สอบถามข้อมูล (Range Query) หาผลรวมของค่าสัญญาณในช่วงพิกัดตั้งแต่ L ถึง R รวมกัน

จงสร้างคลาสโครงสร้างข้อมูล **Segment Tree** แบบดิบขึ้นมาจัดการปัญหานี้โดยห้ามใช้คำสั่งลูปวนบวกตรงๆ ตอนรับ Query

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็มสองตัว $n$ และ $q$ ($1 \le n, q \le 10^5$) แทนขนาดพิกัดโครงข่ายและจำนวนคำสั่งใช้งานตามลำดับ
* **บรรทัดที่ 2 ถึง $q+1$:** คำสั่งในรูปแบบ 1 idx val ($1 \le \text{idx} \le n$) หรือ 2 L R ($1 \le L \le R \le n$)

### ข้อมูลส่งออก (Output)

* แสดงคำตอบของคำสั่งประเภทที่ 2 ในแต่ละบรรทัด

### ตัวอย่าง

**Input**

```
5 5
1 1 5
1 3 2
2 1 3
1 2 10
2 1 3

```

**Output**

```
7
17

```

**คำอธิบาย:** เริ่มต้นอาร์เรย์คือ [0,0,0,0,0] -> อัปเดตพิกัด 1 เป็น 5 ([5,0,0,0,0]) -> อัปเดตพิกัด 3 เป็น 2 ([5,0,2,0,0]) -> ผลรวมช่วง 1 ถึง 3 คือ $5+0+2 = 7$ -> อัปเดตพิกัด 2 เป็น 10 ([5,10,2,0,0]) -> ผลรวมช่วง 1 ถึง 3 คือ $5+10+2 = 17$

## 7. ระบบคำนวณสิทธิ์เข้าถึงทรัพยากร (Product of Array Except Self)

### รายละเอียดโจทย์

คุณมีอาร์เรย์จำนวนเต็ม $nums$ ขนาด $n$ ช่อง ต้องการสร้างอาร์เรย์ผลลัพธ์ $output$ ที่ช่อง $output[i]$ มีค่าเท่ากับ**ผลคูณของตัวเลขทุกตัวในอาร์เรย์เดิม ยกเว้นตัวที่อยู่ที่ดัชนี $i$ ตัวมันเอง**

**เงื่อนไขเหล็ก:** ห้ามใช้โอเปอเรเตอร์การหาร / หรือ // ในโค้ด และห้ามเขียนลูปซ้อนลูปแบบ $O(n^2)$ ระบบต้องประมวลผลเสร็จสิ้นด้วยความเร็ว $O(n)$ และใช้พื้นที่หน่วยความจำคงที่ (ไม่นับอาร์เรย์ผลลัพธ์ที่โจทย์ต้องการ) โดยอาศัยหลักการคำนวณสะสมแบบ Prefix และ Suffix Products

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็ม $n$ ($2 \le n \le 10^5$)
* **บรรทัดที่สอง:** จำนวนเต็ม $n$ ตัวแยกด้วยช่องว่าง (ค่าอาจมีทั้งบวก, ลบ, หรือศูนย์)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม $n$ ตัวในบรรทัดเดียวแยกด้วยช่องว่าง แสดงผลคูณตามเงื่อนไข

### ตัวอย่าง

**Input**

```
4
1 2 3 4

```

**Output**

```
24 12 8 6

```

**คำอธิบาย:**

* ดัชนี 0: $2 \times 3 \times 4 = 24$
* ดัชนี 1: $1 \times 3 \times 4 = 12$
* ดัชนี 2: $1 \times 2 \times 4 = 8$
* ดัชนี 3: $1 \times 2 \times 3 = 6$

## 8. ระบบรวมกลุ่มพันธมิตรเครือข่าย (Disjoint Set Union - DSU)

### รายละเอียดโจทย์

ในการจัดการข้อมูลของกลุ่มผู้ใช้งานบนแพลตฟอร์มคอมมิวนิตี้ขนาดใหญ่ ระบบต้องการกลไกจัดกลุ่มที่รวดเร็ว โดยเริ่มต้นผู้ใช้ทุกคนเป็นอิสระต่อกัน (แยกกันคนละกลุ่ม) ระบบต้องประมวลผลคำสั่ง 2 รูปแบบ:

1. 1 u v: ทำการเชื่อมโยง (Union) ผสานกลุ่มของผู้ใช้ u และผู้ใช้ v เข้าเป็นกลุ่มเดียวกัน
2. 2 u v: สอบถามสถานะ (Find/Check) ว่าผู้ใช้ u และผู้ใช้ v อยู่ในกลุ่มพันธมิตรเดียวกันหรือไม่ (ไม่ว่าจะเชื่อมต่อกันโดยตรงหรือผ่านคนอื่นเป็นทอดๆ)

จงสร้างคลาสโครงสร้างข้อมูล **Disjoint Set Union (DSU)** ที่ใช้เทคนิค Path Compression และ Rank/Size Optimization เพื่อให้การประมวลผลแต่ละคำสั่งมีความเร็วเกบคงที่ ($O(\alpha(n))$)

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็มสองตัว $n$ และ $q$ ($1 \le n, q \le 10^5$) แทนจำนวนสมาชิกและจำนวนคำสั่งตามลำดับ
* **บรรทัดที่ 2 ถึง $q+1$:** คำสั่งในรูปแบบ 1 u v หรือ 2 u v ($1 \le u, v \le n$)

### ข้อมูลส่งออก (Output)

* สำหรับคำสั่งประเภทที่ 2 ให้พิมพ์ YES หากอยู่กลุ่มเดียวกัน หรือ NO หากอยู่คนละกลุ่ม

### ตัวอย่าง

**Input**

```
5 5
1 1 2
1 2 3
2 1 3
2 1 4
1 3 4

```

**Output**

```
YES
NO

```

## 9. ระบบจัดหมวดหมู่รหัสพันธุกรรม (Longest Common Subsequence - LCS)

### รายละเอียดโจทย์

ศูนย์วิจัยชีวสารสนเทศต้องการหารหัสพันธุกรรมร่วมที่ยาวที่สุดระหว่างสาย DNA สองเส้น โดยกำหนดให้ข้อความสตริงมา 2 บรรทัด คือ $S_1$ และ $S_2$

จงหา**ความยาวของสตริงย่อยร่วมที่ยาวที่สุด (Longest Common Subsequence)** ซึ่งสตริงย่อยร่วมนี้ไม่จำเป็นต้องอยู่ติดกันในสตริงเดิม แต่ต้องเรียงลำดับตัวอักษรจากซ้ายไปขวาเหมือนเดิม ให้ใช้ลอจิก Dynamic Programming แบบแจกแจงพิกัด 2 มิติเพื่อแก้ปัญหานี้

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** ข้อความสตริง $S_1$ ($1 \le |S_1| \le 2000$)
* **บรรทัดที่สอง:** ข้อความสตริง $S_2$ ($1 \le |S_2| \le 2000$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงความยาวของ LCS

### ตัวอย่าง

**Input**

```
abcde
ace

```

**Output**

```
3

```

**คำอธิบาย:** LCS คือสตริง ace ซึ่งมีความยาวเท่ากับ 3

## 10. ระบบจัดเส้นทางท่อส่งน้ำมัน (Minimum Spanning Tree - Kruskal's)

### รายละเอียดโจทย์

บริษัทพลังงานต้องการวางแนวระบบท่อส่งน้ำมันเชื่อมต่อระหว่างสถานีจ่ายทั้งหมด $n$ สถานี โดยมีแผนการขุดเจาะถนนเชื่อมระหว่างคู่สถานีที่เป็นไปได้ทั้งหมด $m$ เส้นทาง แต่ละเส้นทางมีค่าใช้จ่ายในการวางระบบต่างกัน

จงเลือกเส้นทางขุดเจาะที่จะทำหน้าที่เชื่อมโยงสถานีทุกสถานีเข้าหากันจนครบ โดยที่**ผลรวมของค่าใช้จ่ายในการวางระบบต่ำที่สุด (Minimum Spanning Tree)** และโครงข่ายต้องไม่มีลักษณะเป็นวงปิด (No Cycles) โดยใช้แนวคิดของ Kruskal's Algorithm ร่วมกับ DSU คัดกรองข้อมูล

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** จำนวนเต็มสองตัว $n$ และ $m$ ($1 \le n \le 10^4$, $1 \le m \le 10^5$) แทนจำนวนสถานีและเส้นทางที่เป็นไปได้
* **บรรทัดที่ 2 ถึง $m+1$:** จำนวนเต็มสามตัว $u, v$ และ $w$ ($1 \le u, v \le n, 1 \le w \le 10^4$) แสดงเส้นทางระหว่างสถานี $u$ และ $v$ ด้วยค่าใช้จ่าย $w$

### ข้อมูลส่งออก (Output)

* จำนวนเต็ม 1 บรรทัด แสดงผลรวมค่าใช้จ่ายที่น้อยที่สุดที่เป็นไปได้ (หากไม่สามารถเชื่อมต่อได้ครบทุกสถานีให้พิมพ์ -1)

### ตัวอย่าง

**Input**

```
4 5
1 2 1
1 3 4
2 3 2
2 4 3
3 4 5

```

**Output**

```
6

```

**คำอธิบาย:** เลือกเส้นทาง $(1, 2)$ cost 1, $(2, 3)$ cost 2, และ $(2, 4)$ cost 3 รวมค่าใช้จ่ายต่ำสุดเท่ากับ $1+2+3 = 6$

## 11. ระบบคำนวณทฤษฎีจำนวนเชิงรุก (Extended Euclidean Algorithm)

### รายละเอียดโจทย์

ในกระบวนการเข้ารหัสลับสไตล์ RSA คอมพิวเตอร์จำเป็นต้องหาค่า **Modular Multiplicative Inverse** ซึ่งก็คือการหาจำนวนเต็ม $x$ ที่ทำให้สมการต่อไปนี้เป็นจริง:

$$a \cdot x \equiv 1 \pmod m$$

กำหนดค่าจำนวนเต็ม $a$ และ $m$ มาให้ จงหาค่า $x$ ที่เป็นจำนวนเต็มบวกที่น้อยที่สุดที่สอดคล้องกับสมการดังกล่าว โดยใช้ **Extended Euclidean Algorithm** เพื่อหาคำตอบด้วยความเร็วระดับ $O(\log(\min(a, m)))$ หากไม่มีคำตอบ (กรณีที่ $a$ และ $m$ ไม่เป็นจำนวนเฉพาะสัมพัทธ์กัน) ให้ระบบตอบกลับเป็น -1

### ข้อมูลนำเข้า (Input)

* จำนวนเต็มสองตัว $a$ และ $m$ แยกกันด้วยช่องว่างในบรรทัดเดียว ($1 \le a, m \le 10^9$)

### ข้อมูลส่งออก (Output)

* จำนวนเต็มบวก $x$ ที่สอดคล้องกับเงื่อนไข หรือตอบ -1 หากไม่สามารถหาค่าได้

### ตัวอย่าง

**Input**

```
3 11

```

**Output**

```
4

```

**คำอธิบาย:** เพราะว่า $3 \times 4 = 12$ และเมื่อนำ $12 \pmod{11} = 1$ พอดี คำตอบของ $x$ จึงเท่ากับ 4

## 12. ระบบหน้าต่างประมวลผลคำสั้นสุด (Minimum Window Substring)

### รายละเอียดโจทย์

กำหนดข้อความสตริงขนาดใหญ่ $S$ และข้อความสตริงเป้าหมาย $T$ มาให้

จงหาช่วงสตริงย่อยของ $S$ (Substring) ที่มีความยาว**สั้นที่สุด** โดยมีเงื่อนไขว่าภายในช่วงสตริงย่อยนั้นจะต้องมีตัวอักษรทุกตัวที่ปรากฏอยู่ใน $T$ ครบถ้วน (นับรวมจำนวนความถี่ซ้ำของอักขระด้วย) ให้แก้ปัญหานี้ด้วยเทคนิค **Two Pointers / Sliding Window** แบบ Dynamic ขยายและหดตัวแปรพิกัดหน้าต่างด้วยความเร็วระดับ $O(|S| + |T|)$ โดยใช้โครงสร้างข้อมูลอาเรย์หรือดิคชันนารีดิบเก็บค่านับความถี่

### ข้อมูลนำเข้า (Input)

* **บรรทัดแรก:** ข้อความสตริง $S$ ($1 \le |S| \le 10^5$)
* **บรรทัดที่สอง:** ข้อความสตริง $T$ ($1 \le |T| \le 10^5$)

### ข้อมูลส่งออก (Output)

* ข้อความสตริงย่อยที่สั้นที่สุดของ $S$ ที่เป็นไปตามเงื่อนไข (หากไม่มีช่วงใดที่ครอบคลุมได้เลยให้พิมพ์สตริงว่าง)

### ตัวอย่าง

**Input**

```
ADOBECODEBANC
ABC

```

**Output**

```
BANC

```

**คำอธิบาย:** สตริงย่อย BANC ที่อยู่ท้ายสตริง $S$ มีตัวอักษร A, B, C ครบถ้วนตามสตริง $T$ และมีความยาวสั้นที่สุดคือ 4 อักขระ
