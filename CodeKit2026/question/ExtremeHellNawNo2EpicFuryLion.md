# Minimum Time to Deliver All Packages

ในโจทย์นี้เรามีเงื่อนไขสำคัญเกี่ยวกับการจัดส่งพัสดุในเครือข่ายเมืองจำนวนหนึ่ง โดยเมืองทั้งหมดมีหมายเลขตั้งแต่ `0` ถึง `n-1` ซึ่งเมือง `0` คือจุดเริ่มต้นของรถส่งของหรือโกดังหลัก

ระบบถนนเชื่อมต่อเมืองต่าง ๆ จำนวน `m` เส้น โดยแต่ละเส้นเชื่อมเมือง `a` กับ `b` และมีค่าเวลาที่ต้องใช้เดินทางเท่ากับ `w` หน่วย ซึ่งการเดินทางระหว่างเมืองทั้งสองสามารถไปมาได้ทั้งสองทิศทาง

ภารกิจหลักคือให้รถส่งของเพียงคันเดียวออกจากเมือง `0` แล้วไปยังเมืองที่ต้องส่งพัสดุทั้งหมดตามรายการที่กำหนด ไม่นับจำนวนพัสดุในแต่ละเมือง เพราะรถสามารถบรรทุกได้ไม่จำกัด แต่ต้องไปเยี่ยมทุกเมืองที่มีพัสดุอย่างน้อยหนึ่งชิ้น

หลังจากส่งพัสดุครบทุกเมืองตามที่กำหนดแล้ว รถไม่จำเป็นต้องกลับมาที่โกดัง และเราต้องหาว่าเส้นทางใดที่จะใช้เวลารวมต่ำที่สุดในการเดินทางผ่านเมืองที่สำคัญทั้งหมด

ด้วยเหตุนี้โจทย์จึงเน้นการวางแผนเส้นทางที่ชาญฉลาด ไม่ใช่การลองทุกทางแบบสุ่ม เพราะจำนวนเมืองและเส้นทางอาจมากจนไม่สามารถคำนวณด้วยวิธี brute-force ได้ในเวลาที่เหมาะสม

---

## Input

```text
n m k
a1 b1 w1
a2 b2 w2
...
am bm wm
p1 p2 p3 ... pk
```

- `n` = จำนวนเมือง
- `m` = จำนวนถนน
- `k` = จำนวนเมืองที่ต้องส่งของ
- ถนนเป็นแบบไปกลับ
- `pi` คือเมืองที่ต้องไปส่งของ

---

## Output

```text
เวลาน้อยที่สุด
```

---

## Example

### Input

```text
5 6 3
0 1 2
0 2 4
1 2 1
1 3 7
2 4 3
3 4 2
1 3 4
```

### Output

```text
10
```

---

## Explanation

เมืองที่ต้องไปคือ `1`, `3`, `4`

เส้นทางที่ดีที่สุดคือ:

```text
0 -> 1 -> 2 -> 4 -> 3
```

ใช้เวลา:

```text
2 + 1 + 3 + 2 = 8
```

แต่ในโจทย์นี้เราต้องคำนวณระยะทางสั้นที่สุดระหว่างจุดสำคัญและหา ordering ที่ดีที่สุดโดยไม่ต้องกลับไปที่โกดัง

แนวคิดที่ใช้: `Dijkstra` + `Bitmask DP`

---

## Constraints

- `1 <= n <= 10^5`
- `1 <= m <= 2 * 10^5`
- `1 <= k <= 15`
- `1 <= w <= 10^9`

---

## Hint

1. เมืองเยอะมาก ห้ามลองทุกเส้นทางตรง ๆ
2. จุดสำคัญมีแค่เมือง `0` และเมืองที่ต้องส่งของ
3. ใช้ `Dijkstra` หา shortest path จากจุดสำคัญแต่ละจุด
4. หลังจากได้ระยะทางระหว่างจุดสำคัญแล้ว ใช้ `Bitmask DP` หา order ที่ดีที่สุด
5. ไม่ต้องกลับเมือง `0` หลังส่งครบ
